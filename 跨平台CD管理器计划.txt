2 现有软件功能要点
2.1 Disk Mediator KUREHA（刻录）
简单刻录接口 — COM 接口提供 Mastering 类，通过属性和方法控制光盘生成。例如 VolumeLabel 属性指定数据轨卷标；InitISOFS 方法选择 ISO 9660 文件系统类型（Level1、DOS兼容、Romeo、Joliet/Joliet+ 等）；AddFile 方法把文件或目录添加到数据轨中；ClearISO、ClearTrack 等方法用于删除文件或音轨。通过这些接口可以创建数据盘、添加音频轨并写入光盘。
多文件系统和多会话支持 — 发布说明及历史记录显示程序支持 Joliet/Joliet+、Rock Ridge 等扩展，以及 Track‑At‑Once/Session‑At‑Once、多会话刻录等高级特性。内置 Momiji 低层引擎用于处理不同驱动器和媒体。
日文 CD‑TEXT 支持 — 历史记录和发布说明强调修正了“仅日文标题的 CD‑TEXT 读取失败”问题，并支持刻录和读取日文 CD‑TEXT，这一点是用户需要保留的关键能力。
2.2 Digital CD Player（播放）
驱动访问方式丰富 — 程序同时支持 Windows ASPI、MS‑DOS MSCDEX、NT/2000/XP 下的直接 SCSI/ATAPI 访问以及即插即用光驱，这使得它不依赖特定驱动层[1]。
音频播放与音质控制 — 支持 jitter 补偿、可调读取速度、50/15 ms 去加重滤波，提供正常播放、单曲/单曲循环、整盘循环和随机播放模式[1]。
CD‑TEXT 支持和远程控制 — 程序可以读取光盘上的 CD‑TEXT 并显示曲目信息；支持 Winamp 低层消息和 Windows 标准 CDPlayer.EXE 命令行选项，从而可与其他软件联动[1]。
3 可用的现代跨平台库
要构建同时支持 Windows 和 macOS 的 CD 管理器，可以利用下列跨平台开源库，它们比旧软件更易于集成并支持现代操作系统：
libburnia / libburn — libburnia 项目提供一组用于刻录光盘的库和命令行工具。其文档指出项目包含 libburn、libisofs、libisoburn 等模块；libburn 是底层编程库，可被第三方应用直接使用来刻录 CD/DVD/Blu‑ray 光盘[2]。它支持擦除/格式化可重写介质、以 Track‑At‑Once 或 Session‑At‑Once 模式刻录数据或音频轨、支持多会话、支持设置刻录速度、扫描总线、获取媒体信息等[3]。这是实现跨平台刻录功能的重要基础。
libcueify — 一个跨平台库，用于提取音频 CD 的 TOC、CD‑TEXT 等元数据并生成 cue 表。项目说明中提到它支持读取 CD‑TEXT，包括 日本 MS‑JIS 编码的 CD‑TEXT[4]。该库可以解决读取日文 CD‑TEXT 的需求。
libcdio / libcdio-utils — GNU libcdio 提供读取 CD‑ROM 的跨平台接口。社区问答指出 libcdio 包含一组命令行工具，它们支持 CD‑TEXT，cd‑info 可以列出光盘信息和曲目标题[5]。利用该库可以扫描光驱、读取 TOC、抓取音频数据以及读取 CD‑TEXT。
这些库均在 Windows、macOS 和各类 Unix 系统上可用，具备活跃维护并支持现代硬件，是替代旧软件核心功能的合适选择。
4 跨平台 CD 管理器的设计方案
4.1 总体架构
核心功能层 – 负责与光驱通信、数据读取/刻录和元数据处理。
刻录模块：封装 libburn 的 API，用于创建数据盘或音频盘，设置文件系统（ISO 9660 Level1/2、Joliet/Joliet+ 等）、多会话控制和写入速度。通过 libisofs 创建 ISO 映像，再调用 libburn/xorrecord 刻录到媒体，实现类似“呉葉”的 Mastering 类功能。
读取模块：利用 libcdio 获取光驱和媒体信息（TOC、track 长度、识别光盘类型），使用 libcueify 提取 CD‑TEXT （尤其是日文 MS‑JIS 编码）并转换为 UTF‑8 显示。对于音频抓取，可使用 libcdio 的 cd-paranoia 或集成 cdparanoia 库实现抖动修正和无损读取，替代旧播放器的 jitter 收集功能。
元数据编码模块：实现 CD‑TEXT 字符编码转换（Shift‑JIS/ISO‑8859‑1 ↔ UTF‑8），支持多语言编辑、字符长度校验等，确保刻录时正确写入日文信息。
音频播放与控制层 – 使用跨平台音频库（如 PortAudio 或 SDL Audio) 播放读取到的音频流，实现普通播放、单曲循环、整盘循环和随机播放。该层还处理播放队列、读取速度调整和去加重滤波等 DSP 功能。
用户界面层 – 提供统一的 GUI，用于浏览光盘内容、编辑 CD‑TEXT、选择刻录选项和控制播放。可选的跨平台方案包括：
Qt（C++ 或 PyQt） – 提供原生外观、良好的国际化支持以及对 macOS/Windows 的成熟部署。Qt 的信号/槽机制易于构建驱动事件响应和进度条。
Electron – 使用网页技术构建界面，适合需要快速开发和跨平台部署，但对硬件访问需通过 Node.js 原生插件包装 libburn/libcdio。
Flutter – 适用于同时支持桌面和移动端；需要通过平台通道调用原生库。
扩展与自动化接口 – 提供命令行接口或脚本 API 以支持批量刻录和第三方集成，类似旧软件的 OLE/Winamp 遥控功能。可以考虑实现 REST API 或 Python 库，允许其他应用调用核心功能。

4.3 工作流程示例
播放光盘：程序启动时通过 libcdio 列出可用光驱→用户选择光驱→读取 TOC 和 CD‑TEXT→显示曲目列表（曲名支持日文）→用户选择播放模式，播放层从光驱抓取音频并输出至音频设备。
抓取音频或制作音轨：读取模块利用 cd-paranoia 将音轨保存为 WAV/FLAC，元数据保存为 cue 文件。编辑模块允许修改曲目信息并重新编码为需要的字符集。
刻录数据盘：用户拖放文件到项目视图→设置卷标、文件系统和刻录模式→程序用 libisofs 构建 ISO 映像并调用 libburn 写入光盘。写入过程中显示进度条及驱动器速度，完成后弹出光盘。
刻录音频盘：用户导入音频文件并编辑曲目信息→程序创建 cue 表并写入 CD‑TEXT（支持日文）→使用 libburn 以 Session‑At‑Once 模式刻录音轨，实现无间隔连续音轨。完成后校验音频完整性。 
5 总结
原始的“呉葉”和“Digital CD Player”都是 Windows 上的旧软件，它们的独特之处在于对 CD‑TEXT（尤其是日文）支持和针对 CD 音频的多样控制。通过分析 KUREHAAPI 文档可知，呉葉提供了 COM 接口和丰富的刻录功能，包括设置文件系统和添加/删除文件；而 Digital CD Player 具有多种读盘接口、抖动处理、读取速度调节和 CD‑TEXT 支持[1]。由于这些程序年代久远且平台受限，直接移植不现实。
新的跨平台 CD 管理器应采用现代开源库，如 libburn（负责跨平台刻录并支持多会话和音频轨刻录[3]）、libisofs 和 libcdio/libcueify（用于读取光盘信息和 CD‑TEXT[5][4]）。通过 C++/Qt 或 Electron 等框架构建图形界面，结合 PortAudio 实现音频播放和控制，同时在编码层确保对日文 CD‑TEXT 的正确处理，就可以在 Windows 和 macOS 上提供类似旧软件的功能，并扩展自动化和脚本接口。
